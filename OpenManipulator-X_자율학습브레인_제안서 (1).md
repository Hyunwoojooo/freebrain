# OpenManipulator-X를 위한 자율 학습 브레인 구축

**옹알이 같은 무작위 움직임에서 시작하여, LLM이 안내하는 목적적 스킬 습득으로 성장하는 로봇 팔 — 이제 기술적으로 구현 가능하다.** 호기심 기반 강화학습과 계층적 LLM 추론, 영속적 메모리 시스템, 그리고 비전-언어 그라운딩을 결합하면, OpenManipulator-X가 자율적으로 작업 공간을 탐색하고, 자기 관절이 무엇을 할 수 있는지 발견하고, 물체를 잡고 놓는 법을 학습하고, 헷갈릴 때는 사람에게 질문하고, 웹 검색으로 새로운 조작 전략을 찾는 시스템을 만들 수 있다. 본 제안서는 RoboGen의 자기 주도 학습 루프, Voyager의 성장하는 스킬 라이브러리, IMGEP의 발달적 커리큘럼, 그리고 최신 VLA 모델을 활용한 구체적이고 단계별 아키텍처를 제시하며, 이 모두를 DYNAMIXEL 기반 하드웨어와 MetaCog-RL 연구 전문성에 맞게 설계했다.

---

## 3계층 인지 아키텍처

시스템의 핵심 설계는 성장하는 아이의 뇌에 대한 발달 신경과학을 모방한다: 빠른 반사 계층이 운동 실행을 담당하고, 중간 수준의 스킬 계층이 행동 레퍼토리를 관리하며, 느린 숙고 계층이 목표와 지식 격차에 대해 추론한다. 각 계층은 서로 다른 시간 스케일과 추상화 수준에서 작동하며, 명확하게 정의된 인터페이스를 통해 소통한다.

**계층 1 — 운동 피질 (1–20 ms, 50–200 Hz).** 이것은 작동 계층이다. ROS 2 Humble 위에서 `ros2_control`과 `dynamixel_hardware_interface` 플러그인으로 실행되며, 관절 위치 또는 속도 명령을 받고 관절 상태(XM430-W350 서보에서 **0.088° 해상도**의 위치, 속도, **2.69 mA/step**의 전류)를 읽어온다. 초기 "옹알이" 단계에서는 이 계층이 원시 관절 명령을 실행한다. 스킬이 발전하면 학습된 정책 — MoveIt 2 모션 계획이나 뉴럴 네트워크 정책(VLA/확산 모델)을 LeRobot의 추론 파이프라인을 통해 실행한다. 이 계층의 하드 안전 래퍼는 관절 한계(하드웨어 정지 전 ±5° 버퍼), 속도 상한(12V에서 최대 46 rpm), 작업 공간 경계, 전류 기반 충돌 감지를 강제한다. DYNAMIXEL의 **전류 기반 위치 제어 모드**는 외부 힘/토크 센서 없이도 유연한 파지를 가능하게 한다.

**계층 2 — 스킬 라이브러리 및 선택기 (10–100 ms).** 이 전술 계층은 Voyager의 절차적 메모리에서 직접 영감을 받아, 검증된 조작 스킬의 성장하는 라이브러리를 유지한다. 각 스킬은 자연어 독스트링, 전제조건, 후제조건, 예상 소요 시간, 성공률 추정치를 가진 Python 함수로 저장된다. 스킬은 `sentence-transformers`를 통한 벡터 임베딩으로 ChromaDB 컬렉션에 인덱싱되어 의미적 검색이 가능하다. 경량 스킬 선택기 — 행동 트리(BehaviorTree.CPP v4) 또는 학습된 어포던스 스코어러(SayCan 스타일 가치 함수) — 가 현재 목표와 세계 상태에 따라 가장 관련 있고 실행 가능한 스킬을 선택한다. 새로운 스킬은 세 가지 채널을 통해 라이브러리에 진입한다: 호기심 기반 RL 발견, LLM 생성 코드(Code-as-Policies 패턴), 그리고 인간 텔레오퍼레이션 시연(기존 GhostArm VR 셋업 활용).

**계층 3 — 인지 집행부 (200 ms–2 s).** 이 전략 계층이 "성장하는 마음"이다. 멀티모달 LLM(API를 통한 GPT-4o 또는 Claude, GPU에서의 로컬 폴백으로 Qwen2.5-VL-7B)을 사용하여 태스크 분해, 목표 생성, 호기심 기반 질문, 새로운 지식을 위한 웹 검색, 반성적 자기 평가를 수행한다. LLM은 작업 메모리 버퍼로부터 구조화된 컨텍스트를 받는데, 여기에는 현재 장면 설명(비전 파이프라인에서), 최근 에피소드 기억, 현재 스킬 레퍼토리 요약, 로봇의 발달 단계가 포함된다. Inner Monologue의 폐루프 아키텍처를 따라, 환경 피드백(성공/실패, 장면 변화, 힘 이상)이 LLM의 컨텍스트에 지속적으로 주입된다.

연구에서 도출된 핵심 통찰은 이 계층들을 **동시에** 구축해서는 안 된다는 것이다. 아이의 인지 능력이 더 단순한 역량 위에 scaffolding 되듯이, 발달 단계를 통해 순차적으로 등장해야 한다.

## 피아제를 모델로 한 발달 단계

시스템은 피아제의 감각운동기 발달 이론에서 직접 영감을 받고, INRIA의 Oudeyer FLOWERS 랩의 발달 로보틱스 연구로 검증된 6단계를 거친다. 각 단계는 이전 단계에서 습득한 스킬을 기반으로 하며, 수동 개입이 아닌 역량 임계값에 의해 자동으로 진행이 트리거된다.

**단계 0: 반사 (1–3일차).** 사전 프로그래밍된 안전 행동과 기본 운동 프리미티브: 관절 위치 제어, 그리퍼 열기/닫기, 홈 위치 복귀. 시스템은 전류 피드백을 모니터링하면서 각 관절을 천천히 스윕하여 자신의 관절 한계를 경험적으로 학습한다. 순운동학은 샘플링된 관절 설정에서 엔드 이펙터 위치(로지텍 웹캠 + ArUco 마커로 추적)를 기록하여 캘리브레이션한다. 이 단계에서 캘리브레이션된 운동학 모델과 검증된 안전 범위가 산출된다. MetaCog-RL 프레임워크가 이 자기 모델에 대한 신뢰도를 메타인지 신호로 모니터링할 수 있다.

**단계 1: 운동 옹알이 (1–2주차).** 무작위 관절 명령으로 작업 공간을 탐색한다. 시스템은 내재적 동기 신호로 **RND (Random Network Distillation)**를 사용한다 — 고정된 랜덤 네트워크가 로봇의 상태(관절 각도 + 엔드 이펙터 포즈)를 특징으로 매핑하고, 학습 가능한 예측기 네트워크가 이 특징을 매칭하려 한다. 높은 예측 오류는 새로움을 신호하여, 에이전트를 탐색되지 않은 구성으로 유도한다. 여기서 ICM 대신 RND를 선택한 이유는 noisy-TV 문제(카메라가 무관한 시각적 노이즈를 보여줄 수 있음)를 회피하고 계산적으로 가볍기 때문이다 — 두 개의 작은 MLP(**2–3 레이어, 256 유닛**)가 단일 GPU에서 수 시간 내에 학습된다. 이 단계의 상태 표현은 순수 자기수용감각: 관절 위치, 속도, 엔드 이펙터 포즈. 훈련은 먼저 OpenManipulator-X의 MuJoCo MJX 병렬 시뮬레이션에서 실행한 후, 보수적 속도 스케일링(시뮬레이션 행동 범위의 30%)으로 실제 하드웨어에 전이된다. 이 단계 끝에서 에이전트는 자신의 역운동학과 도달 가능 작업 공간을 암묵적으로 학습한 상태다.

**단계 2: 물체 발견 (2–4주차).** RGB 웹캠(로지텍 웹캠, eye-to-hand 마운트)이 비전 파이프라인에 본격 투입된다. 깊이 센서가 없으므로, 물체의 3D 위치 추정은 **단안 깊이 추정(Depth Anything V2)** 모델과 카메라 캘리브레이션 기반의 기하학적 투영을 결합하여 수행한다. 또한 작업대에 ArUco 마커를 배치하여 스케일 기준점으로 활용하면 정밀도를 보완할 수 있다. 상태 공간이 Grounding DINO + SAM2 분할을 통한 감지된 물체 위치를 포함하도록 확장된다. 호기심 신호가 순수 RND에서 학습 진행 기반 목표 선택을 가진 **IMGEP (Intrinsically Motivated Goal Exploration Processes)**로 전환된다. 목표 공간은 상호작용 에피소드 후의 물체 위치로 정의된다. 에이전트는 목표(원하는 물체 설정)를 샘플링하고, 이를 달성하려 시도하며, 역량이 가장 빠르게 향상되는 목표에 집중한다 — 자연스럽게 쉬운 것(가까운 물체 밀기)에서 어려운 것(물체를 정확한 위치로 밀기)으로의 커리큘럼을 생성한다. IMGEP는 놀랍도록 효율적이다: Oudeyer 랩에서 Raspberry Pi에서 최근접 이웃 정책으로 실행하며, 리칭 → 밀기 → 파지 진행을 자율적으로 발견하는 것을 시연했다. 4-DOF 팔과 3–5개 물체의 경우, **~500K 시뮬레이션 스텝** (MuJoCo MJX에서 `jax.vmap`으로 4096 병렬 환경 구동 시, RTX 4070으로 ~2–4시간) 내에 의미 있는 물체 상호작용 행동이 나타날 것으로 예상된다.

**단계 3: 파지 및 조작 (4–8주차).** LLM 집행부가 활성화된다. 로봇의 현재 스킬 레퍼토리(리칭, 밀기)와 시각 장면을 관찰한 후 목표를 제안한다: "빨간 큐브를 집어 올려봐." RoboGen 패러다임을 따라, LLM이 새로운 스킬을 위한 보상 함수를 Python 코드로 생성한다. 파지의 경우, 보상은 다음을 결합할 수 있다: (1) 그리퍼와 물체 사이의 거리, (2) 물체 근처에서의 그리퍼 폐쇄력, (3) 파지 시도 후 물체 높이. PPO가 이 LLM 생성 보상으로 MuJoCo MJX 병렬 환경에서 정책을 훈련한다. 중요하게도, LLM은 복잡한 태스크를 서브스킬로 분해하고 서브스킬당 최적의 학습 접근법을 선택한다 — 접촉이 풍부한 파지에는 RL, 장애물 없는 리칭에는 MoveIt 2 모션 플래닝. 성공적으로 학습된 스킬은 검증(물체가 일관되게 목표 높이에 도달하는가?)된 후 코드와 독스트링과 함께 스킬 라이브러리에 추가된다.

**단계 4: 소통적 학습 (8–12주차).** LLM이 **질문하기**와 **웹 검색** 도구를 획득한다. 시스템이 이전에 조작한 적 없는 물체를 만나면(Grounding DINO로 감지되었지만 의미 메모리에 없음), 다음이 가능하다: (a) 인간 오퍼레이터에게 "이 물체가 뭐에요? 어떻게 다루면 되나요?"라고 질문하고 답변을 의미적 지식으로 저장; (b) 검색 API를 통한 function-calling으로 조작 전략을 웹 검색("원통형 물체 파지 방법 로보틱스"); (c) 유사한 과거 상호작용에 대해 에피소드 기억 쿼리. 이는 아이들이 언어를 사용하여 지식을 습득하기 시작하는 발달적 전환을 반영한다. LLM은 도구 사용 프레임워크를 통해 이를 조율한다 — 커스텀 ROS 2 도구가 있는 LangChain이나 GPT-4o와의 직접 function-calling. 웹 검색 결과와 인간 답변은 파싱, 요약되어 출처 귀속과 함께 의미 메모리(ChromaDB)에 항목으로 저장된다.

**단계 5: 자율 실험 (12주차 이후).** 시스템이 이전의 모든 능력을 결합한 개방형 탐색 루프에 진입한다. Voyager의 자동 커리큘럼에서 영감을 받아, LLM이 현재 스킬 라이브러리를 검토하고, 격차를 식별("블록은 집을 수 있지만 쌓는 법은 아직 안 배웠어")하며, 새로운 도전을 제안한다. 시뮬레이션 환경, 보상 함수, 훈련 구성을 자율적으로 생성한다(RoboGen의 제안-생성-학습 주기). 목표 발견을 위해 **비대칭 셀프 플레이**를 사용한다: "앨리스" 정책이 새로운 물체 구성을 만들고 "밥" 정책이 이를 재현해야 하여, 자연스럽게 점점 더 복잡한 조작 도전을 생성한다. 인간 피드백은 시스템이 학습한 것을 보여주고 수정이나 새로운 목표를 요청하는 주기적 체크인을 통해 들어온다.

## 메모리 시스템 설계

메모리 아키텍처는 4개의 영속 저장소와 작업 메모리를 구현하며, 개발용으로 ChromaDB를, 프로덕션으로의 마이그레이션 경로로 FAISS를 사용한다.

**에피소드 기억**은 시간 스탬프가 찍힌 상호작용 기록을 저장한다: 시도한 태스크, 수행한 행동, 센서 읽기, 결과(성공/실패/부분), 시각적 스냅샷. 각 에피소드는 유사성 검색을 위해 벡터로 임베딩된다. LLM이 새로운 태스크를 만나면, 계획에 참고하기 위해 가장 관련 있는 과거 에피소드 5개를 검색한다. 이는 "파국적 망각"을 직접 해결한다 — 에이전트가 새로운 행동을 학습해도 스킬과 경험이 유지된다. 구현: 타임스탬프, 태스크 유형, 성공 불리언, 관련 물체, 발달 단계를 메타데이터 필드로 가진 ChromaDB 컬렉션.

**의미 기억**은 세계에 대한 사실적 지식을 저장한다: 물체 속성(무게, 재질, 취약성), 물리 규칙("유리 물체는 떨어뜨리면 깨진다"), 사용자 선호. 지식은 세 가지 채널을 통해 진입한다: LLM 세계 지식(기본), 웹 검색 결과(런타임), 인간 지시(대화형). 구현: 출처 귀속 메타데이터가 있는 ChromaDB 컬렉션으로, 시스템이 확실한 지식(물리학에서)과 불확실한 지식(단일 웹 검색에서)을 구분할 수 있다.

**절차 기억**은 스킬 라이브러리 자체다: 독스트링의 임베딩이 있는 Python 함수, 학습된 정책을 위한 관련 뉴럴 네트워크 가중치. 구현: 코드/가중치용 파일 시스템 + 인덱싱 및 검색용 ChromaDB.

**공간 기억**은 작업 공간의 영속적 장면 그래프를 유지한다: 물체 정체성, 위치, 공간적 관계가 각 카메라 프레임마다 업데이트된다. 구현: 인식 파이프라인에 의해 업데이트되는 경량 인메모리 그래프(NetworkX), 디스크에 주기적 스냅샷.

**작업 메모리**는 LLM의 컨텍스트 윈도우로, 다음으로 채워진다: 현재 태스크 목표, 최근 에피소드 기억(관련도 상위 5개), 현재 스킬 레퍼토리 요약, 공간 기억 스냅샷, 시스템 상태. 이는 각 LLM 쿼리 전에 구조화된 프롬프트 템플릿으로 조합된다.

## 실제 하드웨어 및 소프트웨어 스택

전체 자재 목록과 소프트웨어 의존성은 서강대학교의 기존 셋업을 기준으로 설계되었다.

**하드웨어.** OpenManipulator-X(**RM-X52-TNM, ~$450**)는 12V에서 **4.1 Nm 정지 토크**, **4096 스텝 절대 엔코더**, 전류 감지를 가진 DYNAMIXEL XM430-W350 서보를 사용하여 4 DOF + 1 그리퍼를 제공한다. 1 Mbps 보드레이트로 U2D2 USB-to-DYNAMIXEL 어댑터를 통해 연결한다. 인식을 위해 **로지텍 웹캠**을 eye-to-hand 구성으로 삼각대 또는 커스텀 브래킷에 마운트한다. RGB 전용 카메라이므로 깊이 정보는 **Depth Anything V2** 단안 깊이 추정 모델로 보완하며, 작업대에 ArUco 마커를 배치하여 스케일 기준점 및 카메라-로봇 좌표계 변환에 활용한다. 1080p@30fps RGB 영상을 제공하며, OpenCV `cv2.VideoCapture`로 직접 접근 가능하다. 컴퓨팅을 위해 **NVIDIA RTX 4070**(12 GB VRAM) 워크스테이션이 MuJoCo MJX GPU 가속 시뮬레이션, VLA 추론(Octo 90M 파라미터 또는 SmolVLA 450M을 4비트 양자화), 로컬 모델의 LLM 추론(4비트 양자화된 Qwen2.5-VL-7B)을 처리한다. RTX 4070의 12GB VRAM 제약 상, 대형 모델은 클라우드 API(GPT-4o, Claude)를 활용하고, 로컬에서는 경량 모델 위주로 운용한다. MuJoCo MJX는 JAX 기반으로 VRAM 효율이 높아, RTX 4070에서도 `jax.vmap`을 통해 4096개 병렬 환경을 구동할 수 있다(Isaac Sim/Lab 대비 VRAM 사용량이 현저히 낮음). 총 추가 하드웨어 비용: OpenManipulator-X 자체 외에 약 **$100 미만**(웹캠 + U2D2).

**소프트웨어 스택.** 미들웨어는 Ubuntu 22.04의 **ROS 2 Humble**로, OpenManipulator-X의 공식 지원이 가장 안정적인 LTS 조합이다. 핵심 ROS 2 패키지: `open_manipulator`(공식 ROBOTIS), `dynamixel_hardware_interface`(ros2_control 플러그인), `moveit2`(모션 플래닝), `usb_cam` 또는 `v4l2_camera`(로지텍 웹캠 ROS 2 드라이버), `ros2_control`과 `joint_trajectory_controller` 및 `gripper_controllers`. 시뮬레이션은 **Gazebo Classic (Ignition Fortress)**를 ROS 2 연동 및 시각화에 사용하며, Humble과의 호환성이 검증되어 있다. GPU 가속 RL 훈련에는 **MuJoCo MJX + MuJoCo Playground**를 사용한다. MJX는 MuJoCo의 JAX 재구현으로, `jax.vmap`을 통해 수천 개의 병렬 환경을 단일 GPU에서 구동하며 VRAM 효율이 뛰어나다(RSS 2025 Outstanding Demo Paper Award 수상). `pip install playground`로 간편 설치 가능하며, Brax의 PPO 구현과 직접 연동된다. OpenManipulator-X의 MJCF(MuJoCo XML) 모델은 기존 URDF에서 변환하거나 직접 작성한다. ML용: 모든 뉴럴 네트워크에 **JAX + PyTorch 2.5+**(MJX 훈련은 JAX, VLA/모방 학습은 PyTorch), 모방 학습 및 VLA 배포용 **LeRobot v0.4**. 단안 깊이 추정용 **Depth Anything V2** (ViT-S 모델 권장, RTX 4070에서 실시간 추론 가능). LLM 통합용: GPT-4o/Claude에 직접 API 호출(레이턴시를 위해 LangChain보다 커스텀 경량 래퍼 선호), 벡터 메모리용 **ChromaDB 0.5+**, 임베딩 생성용 **sentence-transformers**. 호기심 기반 RL용: MJX의 JAX 생태계에 맞춰 **커스텀 RND/ICM 구현**(JAX로 작성, 두 개의 소형 MLP로 구성)을 Brax의 PPO 훈련 루프에 내재적 보상으로 통합. PyTorch 기반 실험에는 **RLeXplore** 라이브러리(Stable-Baselines3 호환)를 보조적으로 활용. 실험 추적은 **Weights & Biases**로.

웹 검색 기능을 위해, LLM의 도구 사용 인터페이스가 웹 검색 API(SerpAPI 또는 Tavily)를 쿼리하고, 관련 페이지를 가져오고, 발견 사항을 요약하는 검색 함수를 노출한다. 이는 LLM의 function-calling 인터페이스에 등록된 간단한 Python 도구로 구현된다 — 무거운 프레임워크 불필요.

## MetaCog-RL과 GhostArm 연결

이 프로젝트는 현우님의 기존 연구를 직접 확장한다. **MetaCog-RL**은 단계 전환과 학습 전략 선택을 통제하는 메타인지 모니터링 프레임워크를 제공한다. 메타인지 컨트롤러는 세 가지 신호를 모니터링한다: (1) 스킬 라이브러리 전반의 학습 진행률(새로운 스킬이 습득되고 있는가?), (2) 순방향 모델의 예측 신뢰도(로봇이 자신의 동역학을 이해하고 있는가?), (3) 태스크 성공률 궤적(성능이 향상되고 있는가 아니면 정체되고 있는가?). 발달 단계의 학습 진행이 포화되면(메타인지 신호), 시스템이 자동으로 다음 단계로 진행한다. 이것은 발달 로보틱스에 대한 메타인지 RL의 자연스러운 적용이며 — 출판 가능한 기여다.

**GhostArm**은 시연 데이터 수집을 위한 텔레오퍼레이션 인프라를 제공한다. VR 텔레오퍼레이션 시연은 LeRobot의 데이터 파이프라인에 직접 공급되어, 모방 학습을 위한 고품질 훈련 에피소드를 생산한다. GhostArm VR 컨트롤러에서 DYNAMIXEL 명령으로의 리더-팔로워 매핑은 이미 해결되었다. LeRobot의 데이터 기록 래퍼(`lerobot-record`)를 추가하면 시연에서 훈련까지의 원활한 파이프라인이 생성된다. 추가로, GhostArm은 human-in-the-loop 수정 메커니즘(TRANSIC 스타일)을 가능하게 한다: 기본 정책이 자율적으로 실행되고, 인간이 VR 관찰을 통해 오류를 감지하면 개입하여 궤적을 수정하여 잔차 수정 데이터를 생성한다.

## 16주 구현 로드맵

**1단계 — 기반 구축 (1–4주차).** 전체 소프트웨어 스택 설정: ROS 2 Humble, OpenManipulator-X URDF→MJCF 변환 후 MuJoCo MJX 환경 구축, Gazebo Classic (Ignition Fortress)을 ROS 2 연동 시각화에 활용, 로지텍 웹캠의 카메라 내부 파라미터 캘리브레이션(OpenCV `calibrateCamera`) 및 ArUco 기반 hand-eye 캘리브레이션. MuJoCo Playground의 manipulation 환경 템플릿을 참고하여 OpenManipulator-X용 커스텀 환경 작성. 안전 래퍼 구현(관절 한계, 속도 상한, 전류 모니터링, 작업 공간 경계). 단계 0(자기 캘리브레이션)과 단계 1(RND를 이용한 운동 옹알이)을 완전히 MJX 시뮬레이션에서 실행. 도메인 랜덤화(`jax.vmap`으로 모델 파라미터를 배치 차원에 걸쳐 랜덤화 — 질량 ±20%, 마찰 ±30%, PD 게인 변동)를 사용하여 기본 리칭 정책의 sim-to-real 전이 검증. 산출물: 팔이 시뮬레이션과 실제 모두에서 자율적으로 작업 공간을 탐색하며, 검증된 운동학적 자기 모델 보유.

**2단계 — 물체 상호작용 (5–8주차).** 인식 파이프라인 통합: 개방형 어휘 물체 감지용 Grounding DINO + SAM2, 3D 위치 파악용 Depth Anything V2 단안 깊이 추정 + ArUco 스케일 보정. 물체 위치 공간에서 학습 진행 기반 목표 선택을 가진 IMGEP 구현. 3–5개의 간단한 물체(큐브, 원통, 구)로 시뮬레이션에서 단계 2(물체 발견)를 실행한 후 실제로 전이. ChromaDB 메모리 시스템(에피소드 + 의미 + 절차) 구현. GhostArm을 통한 파지 태스크 텔레오퍼레이션 시연 수집 시작. 산출물: 팔이 밀기와 사전 파지 행동을 자율적으로 발견; 메모리 시스템이 모든 상호작용을 영속적으로 저장.

**3단계 — LLM 통합 (9–12주차).** 인지 집행 계층 연결: 목표 제안, 보상 함수 생성, 질문, 웹 검색을 위한 function-calling이 있는 GPT-4o API. 검증 파이프라인이 있는 스킬 라이브러리 구현(OpenManipulator-X에 적응된 RoboGen의 제안-생성-학습 주기). MuJoCo MJX 병렬 환경에서 LLM 생성 보상으로 훈련된 단계 3(LLM 가이드 파지) 실행. Inner Monologue의 피드백 주입 패턴 구현 — 각 스킬 실행 후 장면 설명과 성공/실패 신호가 LLM에 피드백. 수집된 시연에서 LoRA를 통해 경량 VLA(SmolVLA 450M 또는 Octo 90M) 파인튜닝. 산출물: 팔이 LLM 생성 커리큘럼에서 파지를 학습하고 다단계 픽 앤 플레이스 태스크를 실행 가능.

**4단계 — 자율 성장 (13–16주차).** 단계 4(소통적 학습)와 단계 5(자율 실험) 활성화. 웹 검색 도구와 인간 Q&A 인터페이스 통합. 자동 목표 발견을 위한 비대칭 셀프 플레이 구현. 단계 전환과 학습 전략 선택을 통제하기 위한 MetaCog-RL 모니터링 프레임워크 연결. 자동 커리큘럼 생성기 구축(Voyager 스타일: LLM이 현재 능력 기반 태스크 제안, 훈련 생성, 결과 검증, 스킬 저장). 종단 평가 수행: 작업 공간에 새로운 물체를 놓고 비발달적 기준선과 비교하여 시스템이 얼마나 빠르게 효과적인 조작 전략을 발견하는지 측정. 산출물: 수 일간의 운영을 통해 자신의 능력을 성장시키는 완전 자율 발달 학습 시스템.

## 기존 접근법과의 비교

이 모든 요소를 결합한 기존 시스템은 없다. **RoboGen**이 자기 주도 제안-생성-학습 주기로 가장 가까우나, 시뮬레이션에서만 작동하고 영속 메모리, LLM 소통, 발달적 단계가 없다. **Voyager**는 스킬 라이브러리 + 자동 커리큘럼 아키텍처를 개척했지만 마인크래프트에만 존재한다. **OpenClaw**의 로보틱스 확장은 LLM-as-brain 제어를 제공하지만 학습 능력이 없다 — 발달적 성장이 아닌 원샷 코드 생성에 LLM을 사용한다. **LeRobot**은 모방 학습 인프라를 제공하지만 자율 탐색이나 호기심 메커니즘이 없다. **ELLMER**은 실제 팔에서 LLM + 힘/비전 피드백을 통합하지만 태스크 특화적이지 발달적이지 않다.

이 시스템의 고유한 기여는 **호기심 기반 발달적 진행**(FLOWERS 랩 연구에서), **LLM 가이드 메타 학습**(보상 생성, 커리큘럼 설계, 지식 습득), **영속적 멀티모달 메모리**(진정한 평생 학습 가능), 그리고 **메타인지 모니터링**(MetaCog-RL 연구에서)을 접근 가능한 하드웨어 위의 단일한 일관된 아키텍처로 통합한 것이다. 이 결합 — 옹알이하고, 발견하고, 조작법을 배우고, 질문하고, 지식을 검색하고, 자신의 학습 진행을 모니터링하는 로봇 — 은 발달 로보틱스와 파운데이션 모델 능력을 융합한 진정으로 새로운 체화 AI 접근법을 대표한다.

## 결론

제안된 시스템은 아키텍처적으로 새롭지만 전적으로 검증된 구성 요소로 구축된다. RND와 IMGEP는 배포 준비가 된 구현이 있는 검증된 탐색 알고리즘이다. LLM 통합은 SayCan, Code-as-Policies, Inner Monologue, RoboGen에 의해 확립된 패턴을 따르며 — 모두 최상위 학회에 발표되었다. OpenManipulator-X는 ROS 2 Humble 지원, LeRobot 통합, MuJoCo MJX 호환성이 확인되었다. 총 추정 하드웨어 예산은 **$550 미만**(팔 + 로지텍 웹캠 + U2D2)이며, 계산 요구 사항(단일 RTX 4070)은 대학 연구실 범위 내다. MuJoCo MJX는 JAX 기반 VRAM 효율이 뛰어나 RTX 4070(12GB)에서도 4096 병렬 환경을 안정적으로 구동할 수 있으며, Isaac Sim/Lab이 요구하는 16GB+ VRAM 제약에서 자유롭다. 16주 타임라인은 풀타임 연구자 1명을 가정한다. 가장 중요한 리스크는 호기심 학습 정책의 sim-to-real 전이(도메인 랜덤화와 보수적 배포로 완화), 실시간 인지적 결정을 위한 LLM 레이턴시(자주 사용되는 쿼리 캐싱과 시간에 민감한 호출에 로컬 VLM 사용으로 완화), 그리고 깊이 센서 없이 단안 깊이 추정에 의존하는 3D 위치 파악의 정확도(ArUco 스케일 보정과 반복 캘리브레이션으로 완화)다. 가장 강력한 출판 가능 기여는 자율 발달 단계 관리를 위한 MetaCog-RL 통합 — 메타인지 모니터링이 로봇 스킬 습득에서 수동 조정된 커리큘럼 스케줄을 대체할 수 있음을 시연하는 것이다.
